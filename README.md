# Trivia Nearby

A mobile-first directory for finding nearby trivia nights at bars and restaurants. Live at **[trivianearby.com](https://trivianearby.com)**.

Enter a city or share your location — get the weekly trivia events happening today, tomorrow, or this week, sorted by distance. Each event card shows the host, time, prize info, and a one-tap link to directions.

## How it works

```
  Scraped data / manual seeds
              │
              ▼
   ingest adapters   ─▶   venues ─▶ enriched via Google Places API
                             │       (lat/lng, photo, address, phone)
                             │
                             ▼
                          events (weekly recurring templates)
                             │
                             ▼
                     event_occurrences  (dated instances,
                             │           generated weekly by cron)
                             ▼
        useVenues hook + PostGIS distance RPC
                             │
                             ▼
                    Mobile-first React UI
```

Users filter by **Today / Tomorrow / This Week**. A Supabase RPC (`get_venues_with_distance`) uses PostGIS to find venues within a configurable radius, then joins against `event_occurrences` for the selected date range.

## Tech stack

- **Frontend**: React 19 + Vite + Tailwind CSS v4 + TypeScript
- **Backend**: Node.js importers + scheduler, Google Places API enrichment
- **Data layer**: Supabase (Postgres + PostGIS + RLS + Storage)
- **Geocoding**: Nominatim (for user-typed locations)
- **Routing**: React Router
- **Tests**: Playwright (e2e)
- **Hosting**: Hetzner VPS, Nginx, Let's Encrypt

## Quick start

Requires Node 20.19+ (see `.nvmrc`) and `pnpm`.

```bash
# Frontend
cd trivia-nearby
pnpm install
cp .env.example .env.local     # fill in Supabase URL + anon key + Places key
pnpm dev                        # http://localhost:5173

# Backend (importers, scheduler, ingest)
cd trivia-backend
pnpm install
cp .env.example .env            # Supabase service role + Places key
```

Full backend operations: see [`trivia-backend/OPERATIONS.md`](trivia-backend/OPERATIONS.md).
Production deployment: see [`trivia-backend/DEPLOYMENT.md`](trivia-backend/DEPLOYMENT.md).

## Seeding a city's worth of venues

The `ingest/seed-city.ts` adapter takes a JSON file of trivia venues and events and:

1. Deduplicates venues (by name + address).
2. Enriches new venues with Google Places data (lat/lng, formatted address, phone, photo).
3. Creates a `trivia_providers` row for the host.
4. Inserts the recurring `events` record and backfills 4 weeks of `event_occurrences`.

```bash
cd trivia-backend
pnpm tsx ingest/seed-city.ts ingest/seeds/philadelphia.json
```

Idempotent — safe to re-run. Example seed: [`trivia-backend/ingest/seeds/philadelphia.json`](trivia-backend/ingest/seeds/philadelphia.json) with 32 hand-curated venues across Center City, Fishtown, South Philly, University City, Manayunk, and nearby suburbs.

## Data model (short version)

| Table              | Role                                                   |
|--------------------|--------------------------------------------------------|
| `venues`           | Location + Google Places metadata (PostGIS point)      |
| `trivia_providers` | Companies that run trivia nights (NerdyTalk, Johnny Goodtimes, etc.) |
| `events`           | Recurring weekly template: venue + day + time + provider |
| `event_occurrences`| Specific dated instance; generated by Sunday scheduler |
| `api_usage_log`    | Google Places rate-limit tracking                      |

The split between `events` (template) and `event_occurrences` (instance) lets providers confirm, cancel, or edit individual weeks without touching the template.

## Project status

This is an active side project — production traffic is modest, but the goal is a directory that's accurate enough for people to actually use. Coverage is strongest in Tennessee (the operator's home base) with Philadelphia recently seeded; other cities are added on demand or via partnership.

**Known rough edges:**

- The authentication layer has legacy code paths from earlier iterations — a consolidation pass is planned (see `ARCHITECTURE_REVIEW.md`).
- The Sunday-night occurrence scheduler has been intermittent; a backfill job (`jobs/backfill-occurrences.js`) is idempotent and safe to run manually.
- Per-event editing by venue owners is scaffolded but not yet complete.
